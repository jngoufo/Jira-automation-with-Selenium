**Project description**<br>This project, "Jira-automation-with-Selenium," is a robust framework designed to automate common user tasks within the JIRA web interface. It leverages Python and Selenium to create script-based test cases, serving as an excellent resource for enhancing technical skills in test automation.<br><br>**Objective**<br>The core objective of this project is to streamline and automate user interaction tests with JIRA, reducing manual effort for repetitive tasks. It adopts a hybrid architectural approach, combining the "Page Object Model" (POM) for better maintainability and readability of test scripts, with a "Data-driven" methodology to allow for flexible test data management. This combination ensures that the automation framework is both scalable and easy to manage.<br><br>**Technologies Used**<br><ul><li><b>Python:</b> The primary programming language used for developing the automation scripts.</li><li><b>Selenium:</b> A powerful tool for automating web browsers, enabling interaction with the JIRA web interface.</li><li><b>Excel:</b> for managing test data in a data-driven approach, allowing test cases to be executed with various inputs stored in spreadsheet format.</li></ul><b>Project Structure</b><br>This repository is organized into several key directories to maintain a clear and modular structure, typical for a Selenium automation framework:<ul><li><i>base</i> includes common librairies or methods used across different parts of the framework.</li><li><i>configfiles</i> includes configuration files for the project, such as test environment settings.</li><li><i>pages</i> implements the "Page Object Model," with each file representing a specific page or component of the JIRA application, containing methods that interact with elements on that page.</li><li><i>reports</i> stores the test reports automatically generated after execution of the test cases.</li><li><i>screenshots</i> stores the screen captures, usually necessary on test failure.</li><li><i>testcases</i> contains the actual test scripts or automation flows that perform actions on JIRA.</li><li><i>testdata</i> stores the Excel or CSV files, depending on whether we use test data in the project.</li><li><i>utilities</i> provides custom scripts that support the test cases.</li><li><i>README.md</i> provides an overview of the project.</li></ul>

**Description du projet**<br>Ce projet, « Jira-automation-with-Selenium », est un cadre robuste conçu pour automatiser les tâches courantes des utilisateurs au sein de l'interface web de Jira. Il exploite Python et Selenium pour créer des cas de test basés sur des scripts, constituant ainsi une excellente ressource pour développer des compétences techniques en automatisation des tests.<br><br>**Objectif**<br>L'objectif principal de ce projet est de structurer et d'automatiser les tests d'interaction utilisateur avec Jira, réduisant ainsi les efforts manuels pour les tâches répétitives. Il adopte une approche architecturale hybride, combinant le « Page Object Model » (POM) pour une meilleure maintenabilité et lisibilité des scripts de test, avec une méthodologie pilotée par les données pour une gestion flexible des données de test. Cette combinaison garantit que le framework d'automatisation est à la fois évolutif et facile à gérer.<br><br>**Technologies utilisées**<br><ul><li><b>Python :</b> Langage de programmation principal utilisé pour développer les scripts d'automatisation.</li><li><b>Selenium :</b> Outil puissant pour automatiser les navigateurs web, permettant l'interaction avec l'interface web de JIRA.</li><li><b>Excel :</b> Pour la gestion des données de test selon une approche basée sur les données, permettant l'exécution de cas de test avec diverses entrées stockées au format tableur.</li></ul><b>Structure du projet</b><br>Ce dépôt est organisé en plusieurs répertoires clés afin de maintenir une structure claire et modulaire, typique d'un framework d'automatisation Selenium :<ul><li><i>base</i> contient les bibliothèques et méthodes communes utilisées dans les différentes parties du framework.</li><li><i>configfiles</i> contient les fichiers de configuration du projet, tels que les paramètres de l'environnement de test.</li><li><i>pages</i> implémente le « Page Object Model », chaque fichier représentant une page ou un composant spécifique de l'application JIRA et contenant des méthodes interagissant avec les éléments de cette page.</li><li><i>reports</i> stocke les rapports de test générés automatiquement après l'exécution des cas de test.</li><li><i>screenshots</i> stocke les captures d'écran, souvent nécessaires en cas d'échec d'un test.</li><li><i>testcases</i> contient les scripts de test ou les flux d'automatisation qui effectuent des actions sur JIRA.</li><li><i>testdata</i> stocke les fichiers Excel ou CSV, selon l'utilisation des données de test dans le projet.</li><li><i>utilities</i> contient les scripts personnalisés qui prennent en charge les cas de test. </li><li><i>README.md</i> fournit une vue d'ensemble du projet.</li></ul>
